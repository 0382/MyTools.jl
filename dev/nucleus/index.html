MyTools.Nucleus.JOrbit — Type
struct JOrbit <: SingleParticleOrbit
    n::Int  # 主量子数
    l::Int  # 轨道角动量
    j::Int  # 总角动量
    tz::Int # 同位旋
end

j-shceme 下的单粒子轨道。该结构体以及下面的MOrbit中， 总角动量和同位旋相关量都是使用物理上量子数的两倍来表示，以回避半整数

MyTools.Nucleus.MOrbit — Type
struct MOrbit <: SingleParticleOrbit
    n::Int  # 主量子数
    l::Int  # 轨道角动量
    j::Int  # 总角动量
    m::Int  # 总角动量的分量
    tz::Int # 同位旋
end

MyTools.Nucleus.NuclearShell — Type
struct NuclearShell
    orbits::Vector{JOrbit}
end

原子核中壳的概念。实际上就是一系列的轨道组合而已，不保证真的具有物理意义。 按照壳模型计算的一般习惯，只需要j-scheme的轨道就够了。

source
Base.merge — Method
merge(ns1::NuclearShell, ns2::NuclearShell)

合并两个壳。反正本代码中，壳只是一个计算上的概念，表示一系列轨道的集合。 把这些集合合并起来就是了。
